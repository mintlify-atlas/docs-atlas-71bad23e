# Quick Start

This guide will walk you through annotating a Solidity contract and generating your first IntentSpec JSON file.

## Prerequisites

- **Node.js** ≥ 18
- A Solidity contract (`.sol` file)

## Step 1: Annotate Your Contract

Add `@custom:agent-*` NatSpec tags to your Solidity contract. Place tags in **block comments** (`/** ... */`) directly above the contract or function.

### Contract-Level Annotations

Add metadata about your contract above the `contract` declaration:

```solidity
/**
 * @custom:agent-version 1.0
 * @custom:agent-description Proxy contract for user operations; only owner or designated user can execute.
 * @custom:agent-invariant owner and user are set at construction and immutable.
 * @custom:agent-invariant Only owner or user can call protected functions unless documented otherwise.
 * @custom:agent-event Executed Emitted when an external call is executed; includes target and success.
 * @custom:agent-event TokensWithdrawn Emitted when ERC20/ERC721/ERC1155 tokens are withdrawn.
 */
contract UserProxy {
    // contract code...
}
```

<Info>
  Contract-level tags are optional but recommended for richer metadata.
</Info>

### Function-Level Annotations

Annotate functions with intent, preconditions, effects, risks, and guidance:

```solidity
/**
 * @dev Withdraw a specific amount of ERC20 tokens
 * @custom:agent-intent Withdraw a specific amount of ERC20 tokens to an address.
 * @custom:agent-precondition Caller is owner or user.
 * @custom:agent-precondition token and to are non-zero; amount > 0 and balance sufficient.
 * @custom:agent-effect Balance of this contract decreases; to receives tokens. Emits TokensWithdrawn.
 * @custom:agent-risk Irreversible transfer.
 */
function withdrawERC20(address token, address to, uint256 amount) external {
    // function implementation...
}
```

<Warning>
  **`@custom:agent-intent`** is **required** for a function to be included in the generated spec. All other function-level tags are optional.
</Warning>

### Available NatSpec Tags

#### Contract-Level

| Tag | Purpose | Example |
|-----|---------|--------|
| `@custom:agent-version` | Contract version | `@custom:agent-version 1.0` |
| `@custom:agent-description` | Short description | `@custom:agent-description Proxy for user ops.` |
| `@custom:agent-invariant` | Invariant (can repeat) | `@custom:agent-invariant owner is immutable.` |
| `@custom:agent-event` | Event name + description | `@custom:agent-event Transfer Token balance change.` |

#### Function-Level

| Tag | Purpose | Example |
|-----|---------|--------|
| `@custom:agent-intent` | **Required.** One-line intent | `@custom:agent-intent Withdraws ERC20 to an address.` |
| `@custom:agent-precondition` | Precondition (can repeat) | `@custom:agent-precondition Caller is owner or user.` |
| `@custom:agent-effect` | Effect (can repeat) | `@custom:agent-effect Balance decreases; emits Transfer.` |
| `@custom:agent-risk` | Risk (can repeat) | `@custom:agent-risk Irreversible transfer.` |
| `@custom:agent-guidance` | Guidance for agents | `@custom:agent-guidance Check balance before calling.` |

## Step 2: Generate the Spec

Run the IntentSpec CLI to extract metadata and generate JSON files.

### Option A: Single File (stdout)

Extract from a single file and print JSON to stdout:

```bash
npx intentspec extract-natspec --file contracts/UserProxy.sol
```

This is useful for:
- Quick inspection
- Piping into other tools (e.g., `jq`)
- CI/CD validation

<Tip>
  Pipe the output to `jq` for pretty formatting: `npx intentspec extract-natspec -f contracts/UserProxy.sol | jq .`
</Tip>

### Option B: Compile Entire Project

Scan your project directory for all `.sol` files and generate JSON files:

```bash
npx intentspec compile
```

This will:
- Recursively scan for `.sol` files (excluding `node_modules` and `.git`)
- Extract metadata from each contract with valid NatSpec
- Write one JSON file per contract to **`intentspec/<ContractName>.json`**

<CodeGroup>

```bash Default (current directory)
npx intentspec compile
```

```bash Custom directory
npx intentspec compile --dir ./src/contracts
```

```bash Short flag
npx intentspec compile -d ./contracts
```

</CodeGroup>

## Step 4: Publish & Use

Now that you have your `intentspec.json` files:

<Steps>
  <Step title="Upload to IPFS or a stable URI">
    Host your spec files at a permanent location (IPFS, Arweave, or HTTPS).
  </Step>
  
  <Step title="Implement IIntentSpec in your contract">
    Add the `IIntentSpec` interface to your contract:
    
    ```solidity
    import "./IIntentSpec.sol";
    
    contract UserProxy is IIntentSpec {
        function getIntentSpecURI() external pure returns (string memory) {
            return "ipfs://QmYourHashHere";
        }
        
        // rest of contract...
    }
    ```
  </Step>
  
  <Step title="Agents discover and validate">
    Agents call `getIntentSpecURI()` to fetch your spec, validate their goals against declared intents, and simulate transactions before signing.
  </Step>
</Steps>

## Example Workflow

Here's the complete workflow from annotation to deployment:

```bash
# 1. Annotate your contract with @custom:agent-* tags
vim contracts/UserProxy.sol

# 2. Generate specs
npx intentspec compile

# 3. Inspect output
cat intentspec/UserProxy.json | jq .

# 4. Upload to IPFS (example using ipfs CLI)
ipfs add intentspec/UserProxy.json
# Returns: QmYourHashHere

# 5. Update your contract to return the IPFS URI
# Implement getIntentSpecURI() -> "ipfs://QmYourHashHere"

# 6. Deploy your contract
forge create UserProxy --constructor-args $OWNER $USER

# 7. Agents can now discover your spec onchain!
```

## Re-generating Specs

After updating your NatSpec annotations, simply run the compile command again:

```bash
npx intentspec compile
```

The CLI will overwrite existing JSON files with the latest metadata.

<Tip>
  Add `intentspec/` to your `.gitignore` if you generate specs as part of your build process, or commit them if you want to track changes to your metadata over time.
</Tip>
