---
title: Function-Level Tags
description: Document function intents, preconditions, effects, risks, and guidance
---

Function-level tags are placed in the block comment directly above each function declaration. They describe what the function does, its requirements, effects, risks, and guidance for agents.

## Available Tags

| Tag | Purpose | Repeatable | Required |
|-----|---------|------------|----------|
| `@custom:agent-intent` | One-line description of function purpose | No | **Yes** |
| `@custom:agent-precondition` | Condition that must be true before execution | Yes | No |
| `@custom:agent-effect` | State change or side effect | Yes | No |
| `@custom:agent-risk` | Potential risk or danger | Yes | No |
| `@custom:agent-guidance` | Guidance for agents calling this function | Yes | No |

<Note>
Only functions with `@custom:agent-intent` are included in the generated spec. All other tags are optional.
</Note>

## @custom:agent-intent

**Required.** A concise, single-line description of what the function does. This is the primary documentation that tells agents the function's purpose.

**Format:**
```solidity
@custom:agent-intent <description>
```

**Example:**
```solidity
/**
 * @custom:agent-intent Withdraw a specific amount of ERC20 tokens to an address.
 */
function withdrawERC20(address token, address to, uint256 amount) external {
  // ...
}
```

**JSON Output:**
```json
{
  "name": "withdrawERC20",
  "signature": "0x44004cc1",
  "intent": "Withdraw a specific amount of ERC20 tokens to an address."
}
```

The `signature` field is automatically calculated as the EVM function selector (first 4 bytes of `keccak256(functionSignature)`).

## @custom:agent-precondition

Describes conditions that must be true before the function can execute successfully. Can be repeated multiple times for different preconditions.

**Format:**
```solidity
@custom:agent-precondition <condition description>
```

**Example:**
```solidity
/**
 * @custom:agent-intent Withdraw a specific amount of ERC20 tokens to an address.
 * @custom:agent-precondition Caller is owner or user.
 * @custom:agent-precondition token and to are non-zero; amount > 0 and balance sufficient.
 */
function withdrawERC20(address token, address to, uint256 amount) external {
  // ...
}
```

**JSON Output:**
```json
{
  "preconditions": [
    "Caller is owner or user.",
    "token and to are non-zero; amount > 0 and balance sufficient."
  ]
}
```

## @custom:agent-effect

Describes state changes, side effects, or events emitted by the function. Can be repeated multiple times.

**Format:**
```solidity
@custom:agent-effect <effect description>
```

**Example:**
```solidity
/**
 * @custom:agent-intent Withdraw a specific amount of ERC20 tokens to an address.
 * @custom:agent-effect Balance of this contract decreases; to receives tokens. Emits TokensWithdrawn.
 */
function withdrawERC20(address token, address to, uint256 amount) external {
  // ...
}
```

**JSON Output:**
```json
{
  "effects": [
    "Balance of this contract decreases; to receives tokens. Emits TokensWithdrawn."
  ]
}
```

## @custom:agent-risk

Documents potential risks, dangers, or irreversible actions. Helps agents understand the consequences of calling the function. Can be repeated multiple times.

**Format:**
```solidity
@custom:agent-risk <risk description>
```

**Example:**
```solidity
/**
 * @custom:agent-intent Withdraw a specific amount of ERC20 tokens to an address.
 * @custom:agent-risk Irreversible transfer.
 */
function withdrawERC20(address token, address to, uint256 amount) external {
  // ...
}
```

**JSON Output:**
```json
{
  "risks": [
    "Irreversible transfer."
  ]
}
```

## @custom:agent-guidance

Provides additional guidance or recommendations for agents calling this function. Use this to suggest best practices, validation steps, or simulation recommendations.

**Format:**
```solidity
@custom:agent-guidance <guidance text>
```

**Example:**
```solidity
/**
 * @custom:agent-intent Execute an arbitrary call to a target contract; may approve ERC20 if needed.
 * @custom:agent-guidance Simulate the call first; ensure target is trusted.
 */
function exec(address target, bytes calldata data, address token, uint256 amount) external {
  // ...
}
```

**JSON Output:**
```json
{
  "agentGuidance": "Simulate the call first; ensure target is trusted."
}
```

## Complete Example

Here's a complete function annotation from UserProxy.sol:

```solidity
/**
 * @dev Execute a call to an external contract
 * @param target The target contract address
 * @param data The call data
 * @param token The token address (address(0) for ETH transactions)
 * @param amount The amount of tokens to check allowance for
 * @custom:agent-intent Execute an arbitrary call to a target contract; may approve ERC20 if needed.
 * @custom:agent-precondition Caller is owner or user.
 * @custom:agent-precondition target is not zero and not this contract.
 * @custom:agent-effect If token and amount set, may approve target to spend tokens.
 * @custom:agent-effect Emits Executed; on failure reverts with target's reason or ExecutionFailed.
 * @custom:agent-risk External call can perform any action; validate target and data off-chain.
 * @custom:agent-guidance Simulate the call first; ensure target is trusted.
 */
function exec(address target, bytes calldata data, address token, uint256 amount) external {
  // ...
}
```

This produces:

```json
{
  "name": "exec",
  "signature": "0x1cff79cd",
  "intent": "Execute an arbitrary call to a target contract; may approve ERC20 if needed.",
  "preconditions": [
    "Caller is owner or user.",
    "target is not zero and not this contract."
  ],
  "effects": [
    "If token and amount set, may approve target to spend tokens.",
    "Emits Executed; on failure reverts with target's reason or ExecutionFailed."
  ],
  "risks": [
    "External call can perform any action; validate target and data off-chain."
  ],
  "agentGuidance": "Simulate the call first; ensure target is trusted."
}
```

## Functions Without Intent

Functions without `@custom:agent-intent` are **excluded** from the generated spec:

```solidity
/**
 * @dev Get ETH balance
 */
function getETHBalance() external view returns (uint256) {
  return address(this).balance;
}
```

This function will **not** appear in the JSON output because it lacks `@custom:agent-intent`.
